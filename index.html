📘 Teams Webhook Setup Guide
Step 1 — Create SharePoint List
Go to your SharePoint site → + New → List → Blank list → Name: BlendERP_Tickets → Create.

Then add these columns exactly (column name | type | notes):

# Column Name (type exactly) Type to select What it stores
autoTitleSingle line of textAlready exists — stores Ticket ID (e.g. 202)
1SubjectSingle line of textTicket subject / issue summary
2Category0Single line of textCategory (IT, Finance, HR, etc.) — note the zero: 0
3SubCategorySingle line of textSub-category (Software Bug, Leave Request, etc.)
4PrioritySingle line of textUrgent / High / Medium / Low
5StatusSingle line of textOpen / In Progress / On Hold / Pending Closure / Closed
6RaisedBySingle line of textFull name of person who raised the ticket
7MobileSingle line of textRequester mobile number (+91-XXXXXXXXXX)
8RequesterEmailSingle line of textEmail of person who raised the ticket
9AssignedTo0Single line of textAssignee name(s) — note the zero: 0
10AssigneeEmailSingle line of textAssignee email(s)
11TargetDateDate and TimeTarget resolution date (optional)
12RaisedOnDate and TimeDate ticket was submitted
13ClosedOnDate and TimeDate ticket was closed (blank until closed)
14DescriptionMultiple lines of textFull ticket description
15ResolutionMultiple lines of textResolution notes when closing / change notes when updating
16SourceSingle line of textnew or imported (legacy tickets)
17EventTypeSingle line of textNew Ticket / Updated / Resolved / Closed / Reminder
⚠ Important: Category0 and AssignedTo0 must be typed with a zero (0) at the end — not the letter O. SharePoint reserves the names Category and AssignedTo internally.

Step 4 — Teams Channel Webhook (for instant notifications)
Open Teams → go to your support channel → click … → Connectors → Incoming Webhook → Configure.
Name it: Blend ERP Support | Upload a logo (optional) | Click Create.
Copy the URL and paste in the Teams Webhook URL field above.
Now every ticket action posts a card to your Teams channel automatically — no Power Automate needed.

Step 5 — Power Automate Standard flows (for email notifications — free)
In Power Automate → New flow → Automated cloud flow.
Trigger: SharePoint — When an item is created (Standard — free) → select your BlendERP_Tickets list.
Action: Outlook — Send an email (V2) → To: RequesterEmail → Subject: Ticket #[Title] raised → Body: fill with SP columns.
Save. This flow runs free whenever the app creates a new item in the list.

Step 6 — GitHub Pages + Teams Tab
Rename this file to index.html → push to GitHub repo → Settings → Pages → Source: main/root.
Add as Teams tab: Channel → + → Website → paste your GitHub Pages URL.

Security: Bearer token is stored in this browser only (localStorage). It is never uploaded to GitHub. Rotate it monthly from the Azure portal.
